Major Projects
  • CommitStream Azure-cloud-based Service Led architecture and development of CommitStream, an Azure-hosted microservice built with Node.JS, Express, mocha, EventStore, AngularJS, JSON, JSON Schema, and Hypertext Application Language (HAL). System comprised of auto-scaling App Services and 3 database VMs running EventStore in Clustered mode within an Availability Set and a Load Balancer. We host the code in GitHub and utilize continuous deployment based on Azure/GitHub code push deployments. The services listens for version control system commit event messages delivered by WebHooks or HTTP clients from cloud-based or on-premise servers. It hosts plugins, developed in-house and by third-party company contributors, that translate native event payloads into a common format. These events get summarized, stored, aggregated by work item, team, and program. The summaries surface within item detail and TeamRoom panes inside VersionOne's flagship agile ALM product. This allows large enterprises with teams that work with heterogeneous VCS tools and repos to quickly link their code changes to IDs of planned work items. It facilitates better daily agile/Scrum standup conversations. Worked with a team of local developers in Atlanta/Alpharetta and two remote developers in Argentina over nine months to develop and deploy into production. Our team actively updates, monitors, and supports this system via Pingdom, Pendo usage analysis, ZenDesk ticketing, and with Azure logging tools.
  • Bulk Web API Implementation Led development of new Bulk API endpoint for the VersionOne API using C# 6, F# 4.1, YAML & JSON, JetBrains Rider IDE, Node.js and TypeScript with the Ava test runner for end-to-end acceptance test automation. VersionOne's legacy REST API only supports the creation, modification, or operation-invocation of a single system entity per HTTP request. The Bulk API allows creating multiple new entities in one request, modifying multiple entities, or invoking operations on multiple entities via SQL-like queries. A single HTTP command can combine create, update, and execute commands. Adoption of this feature has increased performance speeds dramatically. As one example, bulk deleting of 2,000 expired messages takes less than two minutes now compared to upwards of 20 minutes with the leagcy endpoint.
  • Slack Plugin Led design of Slack plugin for VersionOne that responds to in-app domain events and formats them for delivery to Slack Incoming WebHook endpoints to notify members about relevant work item events happening inside VersionOne. Built with C# 6, Visual Studio 2015, NUnit, StructureMap, and Log4Net. Collaborated with remote team in Argentina who completed features for the plugin.
  • Feature Plugins Support Led project to introduce modular/pluggable feature support into VersionOne's ALM product. Accomplished this by separating interface/contract definitions of highly stable abstractions from concrete implementations and identifying extension points within the system's API routing and UX rendering sub-components. Discrete features get versioned and packaged as single DLLs (with module-specific views, APIs, DTOs, etc) capable of run-time deployment & hot-swapping. This led to ability of feature teams to independently develop and test their features without the common branch and merge issues plaguing many monolithic codebases. Test engineers also now create test suites against the API layer and module contracts without being tightly-coupled to all implementation details.
